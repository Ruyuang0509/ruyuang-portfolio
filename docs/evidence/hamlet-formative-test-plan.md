# AI 文學故事 MV：形成性評估計畫

## 狀態與邊界

- 狀態：`planned`。
- 此文件定義未來測試任務、要蒐集的證據與修訂用途；目前沒有參與人數、執行日期、觀察紀錄或學習結果。
- 公開案例必須維持 `testing.statusKey = notValidated`，直到真實 study record、限制與證據檔完成核對。

## 評估問題

1. 學習者能否依八幕順序重述《Hamlet》的背景、衝突與結局？
2. 以 B1 為目標、每句 12–18 words 的字幕，在一般播放速度下是否可讀？
3. 教師是否認為文學事實、任務難度與媒體主次適合課堂使用？

## 預定參與角色

- 大學通識英文學習者。
- 授課教師或教學助理。

實際人數、招募條件與日期必須在執行前另行確認，不在此先填入。

## 任務與修訂用途

| 任務 ID | 預定任務 | 蒐集證據 | 設計決策用途 |
|---|---|---|---|
| `story-retell` | 觀看後依序重述背景、衝突與結局 | 順序錯置、人物誤解、遺漏的關鍵事件 | 修訂場景顆粒度與因果提示 |
| `caption-readability` | 播放時閱讀字幕並說明每幕事件 | 未讀完的字幕、詞彙問題、換行問題 | 調整字數、目標難度與每幕停留時間 |
| `teacher-review` | 檢視原作正確性、任務適配度與媒體主次 | 事實修正、教學引導、評量建議 | 修訂 Prompt 人工核對點與教師／學生指南 |

## 建議流程

1. 執行前確認同意方式、紀錄範圍與資料保存位置。
2. 先完成個別觀看與任務，不在過程中提示正確答案。
3. 記錄可觀察問題與原話；解讀與推論分開保存。
4. 教師評閱在學習者任務後進行，避免先用教師意見取代實際可讀性觀察。
5. 依任務 ID 建立修訂清單，保留版本、原因與修改後再測結果。

## 空白 study record 最低欄位

- Study ID 與版本。
- 執行日期、角色與實際人數。
- 任務、觀察方式與同意狀態。
- 原始觀察摘要與證據路徑。
- 限制、偏差與不能推論的事項。
- 對應的設計修改與決策理由。

## 狀態升級門檻

只有在上述 study record、真實證據、限制與修訂紀錄都存在時，才能評估是否把 `notValidated` 改為 `exploratory`。單次教師意見、非正式聊天或預先寫好的預期結果都不能當成已驗證學習成效。
